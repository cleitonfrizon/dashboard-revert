## User Story

Como **Cleiton**, eu preciso ver **gasto e conversões da Google Ads da Revert ao lado da Meta no mesmo dashboard**, para que **eu compare ROI por canal em 30 segundos e identifique onde realocar verba**.

---

## Acceptance Criteria

1. Bloco G — Google Ads renderiza no dashboard com:
   - Totais de spend 30d e conversions 30d em destaque
   - Tabela com top 8 campanhas (nome, spend, CTR, conversões, CPL real)
2. Cache n8n popula `google_ads.campanhas` + `google_ads.totals` consistentes com a Google Ads API
3. `sources_status.google_ads = 'ok'` quando a chamada da última hora foi bem-sucedida; `'stale'` se a última fetch tem mais de 1h; `'error'` em caso de falha; `'not_configured'` se variáveis de ambiente ausentes
4. Match com Reonic via UTM (`utmSource=google_ads` + `utmCampaign`) preenche `leads_reonic` e `cpl_real` por campanha
5. CORS, autenticação, performance (P95 < 9s) e CPL real ≤ 15% off da Meta (sanity check) mantidos
6. Story 1.6 (Bloco C — Campanhas) ganha coluna "Canal" (`Meta` / `Google`) ou bloco G separado mantém o setup atual (decisão durante implementação)

## Arquitetura

```
Cron 30min
```

**Tipo do novo nó:** `n8n-nodes-base.code` (porque a Google Ads API exige OAuth com refresh dinâmico — node httpRequest puro não dá conta).

### Pseudo-código do nó "Buscar Google Ads"

```javascript
// 1) Renovar access_token via refresh_token
const tokenRes = await this.helpers.httpRequest({
  method: 'POST',
  url: 'https://oauth2.googleapis.com/token',
  body: {
    client_id: $env.GOOGLE_ADS_CLIENT_ID,
    client_secret: $env.GOOGLE_ADS_CLIENT_SECRET,
    refresh_token: $env.GOOGLE_ADS_REFRESH_TOKEN,
    grant_type: 'refresh_token',
  },
  json: true,
});

// 2) GAQL searchStream
const customer = $env.GOOGLE_ADS_CUSTOMER_ID;
const query = `SELECT campaign.id, campaign.name, campaign.status,
  metrics.cost_micros, metrics.impressions, metrics.clicks,
  metrics.ctr, metrics.conversions, metrics.cost_per_conversion
  FROM campaign WHERE segments.date DURING LAST_30_DAYS
  AND campaign.status = 'ENABLED'
  ORDER BY metrics.cost_micros DESC LIMIT 50`;

const adsRes = await this.helpers.httpRequest({
  method: 'POST',
  url: `https://googleads.googleapis.com/v17/customers/${customer}/googleAds:searchStream`,
  headers: {
    Authorization: `Bearer ${tokenRes.access_token}`,
    'developer-token': $env.GOOGLE_ADS_DEVELOPER_TOKEN,
    'login-customer-id': $env.GOOGLE_ADS_MCC_ID,
  },
  body: { query },
  json: true,
});

// 3) Achatar response
const rows = (adsRes[0]?.results || []).map(r => ({
  id: r.campaign.id,
  name: r.campaign.name,
  status: r.campaign.status,
  spend: Number(r.metrics.costMicros || 0) / 1_000_000,
  impressions: Number(r.metrics.impressions || 0),
  clicks: Number(r.metrics.clicks || 0),
  ctr: Number(r.metrics.ctr || 0),
  conversions: Number(r.metrics.conversions || 0),
  cost_per_conversion: Number(r.metrics.costPerConversion || 0) / 1_000_000,
}));

return [{ json: { campanhas: rows, customer_id: customer } }];
```

### Alteração no `Calcular Metricas`

Adicionar match Google Ads ↔ Reonic e popular `cache.google_ads`:

```javascript
const googleAdsRaw = safeFirst('Buscar Google Ads') || {};
const googleCampanhas = (googleAdsRaw.campanhas || []).map(c => {
  const leadsG = contactsWithResponse.filter(co =>
    String(co.utmSource || '').toLowerCase().includes('google')
    && matchesCampaign(co, c.name)
  );
  return {
    ...c,
    leads_reonic: leadsG.length,
    cpl_real: leadsG.length > 0 ? c.spend / leadsG.length : 0,
  };
});

cache.google_ads = {
  customer_id: googleAdsRaw.customer_id || '',
  campanhas: googleCampanhas,
  totals: {
    spend_7d: 0,   // TODO: query separada com LAST_7_DAYS
    spend_30d: googleCampanhas.reduce((s,c)=>s+c.spend, 0),
    conversions_7d: 0,
    conversions_30d: googleCampanhas.reduce((s,c)=>s+c.conversions, 0),
  },
};
cache.meta.sources_status.google_ads = googleCampanhas.length ? 'ok' : 'error';
cache.meta.sources_status.last_google_fetch = nowIso;
```

## Riscos / Tradeoffs

| Risco | Mitigação |
|---|---|
| `developer_token` aprovação demorar | Iniciar processo cedo; placeholder atual mostra "Aguardando configuração" sem quebrar UX |
| Refresh_token expirar (Google revoga após 6 meses sem uso) | Cron usa diariamente → não expira; documentar em ADR-032 |
| Match UTM falha se GTM não tem `utm_source=google_ads` | Q-7 valida antes; fallback: usar `utm_source=google` |
| Volume de campanhas explodir (50+) | Query já limita a 50; paginação via `pageToken` se necessário (segunda fase) |
| Custo extra de chamadas Google Ads | API tem free tier alto; cron a cada 30min = ~1500 chamadas/mês, dentro do limite |
